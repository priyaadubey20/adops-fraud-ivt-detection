# Ad Fraud & Invalid Traffic (IVT) Detection Pipeline

## 📌 Project Overview
In programmatic advertising, advertisers expect their messages to reach real audiences. Traffic generated by automated scripts, scrapers, and bot networks compromises system trust and creates artificial revenue spikes that ultimately harm supply platform value.

This repository hosts a data validation pipeline that groups interaction records by standard network identifiers, calculates total traffic velocity, and leverages conditional classification flags to segment invalid traffic streams effortlessly.

---

## 🛠️ Tech Stack & Database Architecture
- **Language:** SQL (Structured Query Language)
- **Concepts Applied:** Text Pattern Matching (`LIKE`), Conditional Filtering (`CASE WHEN`), Log Data Aggregation (`GROUP BY` / `HAVING`).

The underlying architecture maps raw incoming web click data to a core logging registry:
- `Raw_Traffic_Logs`: Stores atomic session timestamps, network points, and client user-agent strings.
